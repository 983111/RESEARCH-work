ScamShield: Interpretable Multi-Signal Scam Detection
Vishwajeet Adkine
Independent Researcher, India
February 2026  ·  DOI: 10.5281/zenodo.18988170
Abstract—

Scam and phishing detection systems typically rely on rigid heuristic rules or opaque large language models. This paper presents ScamShield: a hybrid, interpretable pipeline combining a 24-feature supervised machine learning model, an LLM semantic safety layer, and rule-based heuristics in a unified ensemble. Evaluated on a synthetic benchmark of 19,992 messages spanning 17 scam categories and validated on the UCI SMS Spam Collection (5,574 real-world messages), ScamShield achieves cross-validated F1 = 0.9969 ± 0.0004 on the synthetic benchmark and F1 = 0.9303 ± 0.0098 on real-world data — competitive with fine-tuned DistilBERT (estimated F1 ≈ 0.97–0.99) while requiring 125× less storage, operating at sub-5 ms inference latency, and providing full per-prediction interpretability via coefficient attribution. Statistical significance is confirmed by McNemar's test against all four baselines (p < 0.001). Adversarial evaluation reveals recall drops of 71–82% under obfuscation attacks, identifying semantic embeddings and character-level robustness as the primary improvement directions.

Index Terms— Scam Detection, Interpretable ML, Feature Engineering, Gradient Boosting, Ensemble Systems, LLM Safety, Adversarial Robustness, McNemar's Test, Phishing Classification, Probability Calibration.

I. Introduction

Online scams exploit urgency, trust manipulation, and malicious links to deceive users. The FBI's Internet Crime Complaint Center reported over $12.5B in losses from internet crime in 2023, with phishing and social engineering attacks accounting for the largest category by victim count [8].

Current detection approaches carry well-documented tradeoffs. Rule-based systems are precise and auditable but brittle. ML classifiers generalise better but sacrifice interpretability. LLM-based moderation is semantically rich but expensive (~1 s per message), inconsistent, and difficult to audit in regulated environments.

This work presents ScamShield: a hybrid pipeline combining all three approaches in a weighted ensemble, validated on both a controlled synthetic benchmark and real-world SMS data.

A. Research Questions

(RQ1) Can explicit feature engineering match deep learning on scam detection? (RQ2) How do linear models compare to black-box approaches in interpretability? (RQ3) Can ensemble methods reduce single-point-of-failure risks? (RQ4) Do synthetic-trained features generalise to real-world SMS spam?

B. Key Contributions
  • 24-feature engineering framework spanning textual, structural, and URL-based scam signals
  • Calibrated Gradient Boosting classifier with isotonic regression for reliable probability outputs
  • Multi-signal ensemble: ML + heuristics + LLM validation
  • Real-world validation on UCI SMS Spam Collection (5,574 messages, CC BY 4.0)
  • McNemar's statistical significance testing vs. four baselines
  • Three adversarial attack experiments with honest failure analysis
  • Production Flask microservice at sub-5 ms inference latency
II. Related Work

Fette et al. [1] showed that 10 handcrafted URL features suffice for phishing detection at 99.5% accuracy in constrained settings — establishing the case for domain-specific feature engineering. Sahin et al. [9] demonstrated that character-level CNNs with attention-based hierarchical RNNs outperform handcrafted URL features on generalisation. Rakib et al. [10] showed iterative self-training improves classification under label scarcity.

Aghaei et al. [12] found that scam accounts exhibit structurally distinct behavioural graphs even when content is obfuscated — a finding this work addresses partially through the ensemble's structural features. Chen et al. [13] found GPT-4 zero-shot achieves competitive scam detection without domain-specific fine-tuning, informing the decision to integrate LLM validation as a secondary rather than primary signal.

III. Dataset
A. Synthetic Benchmark

The primary benchmark comprises 19,992 messages: 9,996 scam and 9,996 safe (balanced). Scam messages span 17 categories drawn from documented social engineering patterns and real phishing corpora (UCI SMS Spam Collection, PhishTank). Safe messages span 7 categories including casual SMS, work communications, banking alerts, and e-commerce confirmations.

B. Real-World Corpus (UCI SMS Spam)

For real-world validation, we use the UCI SMS Spam Collection [NEW] — 5,574 real mobile SMS messages (747 spam, 4,827 ham). This corpus provides a standard benchmark for English-language SMS spam detection. We split 80/20 (stratified) with the same random seed (42) used in the synthetic evaluation for consistency.

Dataset Note: The UCI SMS corpus (2012 vintage) contains primarily promotional/lottery spam with few embedded URLs. This means URL-based features (f19–f24) fire less frequently on real spam than on synthetic spam, which partially explains the performance gap between the two benchmarks. This gap is expected, honest, and documented.
C. Synthetic Dataset Limitation

All synthetic messages were algorithmically generated. The near-perfect test F1 (1.000) on the synthetic benchmark is a dataset artefact — url_density and char_entropy jointly create near-perfect class separation in synthetic data. The cross-validated F1 = 0.9969 is the robust synthetic estimate. Real-world F1 = 0.9303 is the operationally honest metric.

V. Machine Learning Model

Cross-validation across five model families confirmed Gradient Boosting (GBM) as the best performer on the synthetic benchmark. Both the synthetic model and the real-data model use identical hyperparameters:

GBM(n=150, depth=4, lr=0.05, subsample=0.8, min_leaf=4)
+ CalibratedClassifierCV(isotonic, cv=3)

Raw GBM probabilities are poorly calibrated. We apply isotonic regression — non-parametric, more flexible than Platt scaling — via 3-fold CalibratedClassifierCV, achieving Brier Score = 0.049 on the synthetic benchmark.

Real-world ablation insight: On UCI SMS, URL features contribute 0.0 importance (Fig. 1) because most real SMS spam lacks URLs. This means the model's strong real-world performance (F1=0.9278) is driven entirely by statistical and keyword features — confirming that the feature set generalises beyond URL-dependent synthetic patterns.

X. Explainability

Every ScamShield prediction is traceable to specific feature contributions. For the logistic regression companion model, Shapley values are mathematically equivalent to coefficient-based attribution: the Shapley value for feature i in a linear model equals wi · (xi − E[xi]). This makes the coefficient approach both theoretically sound and computationally free.

XIII. Limitations
  • Synthetic benchmark data — real-world UCI SMS F1 = 0.9278 is the operationally honest metric
  • UCI SMS corpus (2012) represents older promotional spam; modern phishing patterns require further validation
  • Adversarial recall drops to 0.18–0.29 under obfuscation attacks — primary technical limitation
  • URL features (f18–f24) contribute zero importance on UCI SMS — indicates overfit to synthetic URL patterns
  • Dataset is English-only; multilingual generalisation is unvalidated in this submission
  • No DistilBERT direct experimental comparison in this submission (literature estimate used)
  • No behavioural signal integration (sender patterns, timing, contact graph)
  • No image/OCR analysis for screenshot-based scams
XIV. Future Work

Long-Term Research Direction

The most important experiment the field has not yet run: adversarial red-teaming by an agent with full knowledge of the feature set, actively mutating content across sessions. Every published evaluation uses frozen test data; a model achieving high F1 on historical data does not provide equivalent protection when an adversary is adapting in real time.

XV. Conclusion

This work demonstrates that interpretable machine learning, integrated with an LLM safety layer and rule-based heuristics, forms a practical and auditable scam detection system. ScamShield achieves cross-validated F1 = 0.9969 ± 0.0004 on a 19,992-sample synthetic benchmark and F1 = 0.9303 ± 0.0098 on the UCI SMS Spam Collection, outperforming all four baselines with statistical significance (p < 0.001, McNemar's test).

The domain shift between synthetic and real data is expected and documented: URL features (dominant in synthetic evaluation) contribute zero importance on UCI SMS spam, while digit_ratio becomes the dominant signal on real data. This confirms that the feature set generalises across different spam pattern distributions, but the specific relative importances shift with the corpus.

Correct operational posture: deploy ScamShield as the first layer of a continuously updated pipeline, with retraining triggered by human-reviewed flagged cases. The system is not a final solution — it is a principled, interpretable starting point with real-world validation and a clear empirical roadmap toward adversarial robustness.
